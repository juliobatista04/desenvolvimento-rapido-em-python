O que é Streamlit e por que usá-lo?

Imagine poder construir um dashboard interativo — com gráficos, filtros, tabelas e downloads — usando apenas Python. Sem configurar servidores, sem escrever HTML, sem aprender JavaScript. Isso é exatamente o que o Streamlit oferece.

O Streamlit é uma biblioteca open-source criada em 2019 que transforma scripts Python em aplicações web em minutos. Ele foi projetado especificamente para cientistas de dados e analistas que precisam compartilhar resultados de forma visual sem depender de um time de desenvolvimento web.

⚡

Rapidez

Do script ao app em minutos. Salva o arquivo, o browser atualiza.

🐍

Só Python

Sem HTML, CSS, JS. Você já sabe tudo que precisa para começar.

🔁

Reatividade

Qualquer interação do usuário re-executa o script automaticamente.

📦

Ecossistema

Integra nativamente com pandas, plotly, scikit-learn, e mais.

Como o Streamlit funciona?

O mecanismo central é simples: sempre que o usuário interage com algum widget (clica num botão, move um slider), o Streamlit re-executa o script Python do topo ao fundo. O estado atual dos widgets é preservado entre as re-execuções. Isso cria a sensação de interatividade.

💡
Analogia para fixar

Pense no Streamlit como um "Excel interativo" em Python: cada vez que você muda um valor em uma célula (widget), a planilha recalcula (re-executa o script). Mas em vez de fórmulas, você usa Python.

Instalando

TERMINAL 
# Instalar o Streamlit
pip install streamlit pandas numpy

# Criar um arquivo e rodar
echo "import streamlit as st; st.write('Olá, mundo!')" > app.py
streamlit run app.py

O Streamlit abrirá automaticamente no seu browser em http://localhost:8501.

⚠️
Atenção

Use sempre streamlit run app.py para rodar, nunca python app.py. O segundo não vai renderizar a interface web.

Configuração da Página e Título

Todo app Streamlit começa com uma chamada de configuração. Vamos olhar as primeiras linhas do nosso arquivo app.py:

PYTHON — app.py 
import streamlit as st
import pandas as pd
import numpy as np

# CONFIGURAÇÃO — deve ser a PRIMEIRA chamada st.
st.set_page_config(
    page_title="Dashboard de Funcionários",
    layout="wide"
)

st.title("📊 Dashboard de Análise de Funcionários")

Entendendo cada linha

Parâmetro O que faz Opções
page_title Define o título da aba do browser Qualquer string
layout Controla a largura do conteúdo "centered" (padrão) ou "wide"
page_icon Ícone da aba do browser Emoji ou caminho de imagem
initial_sidebar_state Estado inicial da sidebar "auto", "expanded", "collapsed"

Elementos de texto básicos

O Streamlit oferece várias funções para exibir texto, do simples ao formatado:

PYTHON 
st.title("Título grande")          # H1
st.header("Cabeçalho")             # H2
st.subheader("Sub-título")         # H3
st.write("Texto comum")            # write() é o canivete suíço do st.
st.markdown("**Negrito** e *itálico*")  # Suporta Markdown
st.caption("Texto pequeno / nota")   # Texto de legenda
st.code("print('olá')", language="python") # Bloco de código
🎯
Dica do instrutor

O st.write() é "mágico": ele detecta o tipo do dado e renderiza automaticamente. Passe um DataFrame e ele vira tabela. Passe um dict e vira JSON. Passe um gráfico Plotly e ele renderiza. É sempre um bom ponto de partida.

Carregando e Preparando Dados

A parte mais importante de qualquer dashboard é a origem e qualidade dos dados. No nosso app, os dados são gerados programaticamente — mas a mesma estrutura serve para carregar um CSV, consultar um banco de dados ou chamar uma API.

PYTHON — app.py 
@st.cache_data                   # ← DECORATOR MÁGICO — veremos abaixo
def carregar_dados():
    dados = {
        "nome":    ["Ana", "Bruno", "Carlos", "Daniela", "Eduardo"],
        "idade":   [23, 35, 29, np.nan, 40],
        "cidade":  ["SP", "RJ", "SP", "MG", "RJ"],
        "salario": [3000, 5000, 4000, 3500, np.nan],
        "data_contratacao": pd.to_datetime([...])
    }

    df = pd.DataFrame(dados)

    # ── LIMPEZA DE DADOS ──────────────────────────────
    df["idade"]   = df["idade"].fillna(df["idade"].mean())
    df["salario"] = df["salario"].fillna(df["salario"].median())

    # ── FEATURE ENGINEERING ───────────────────────────
    df["salario_anual"]   = df["salario"] * 12
    df["ano_contratacao"]  = df["data_contratacao"].dt.year
    df["categoria_salario"] = df["salario"].apply(
        lambda x: "Alto" if x > 4500 else "Médio" if x > 3000 else "Baixo"
    )

    return df

df = carregar_dados()

O @st.cache_data — o segredo da performance

Lembra que o Streamlit re-executa o script inteiro a cada interação? Imagine carregar um CSV de 1 GB toda vez que o usuário move um slider. Seria lentíssimo.

O decorador @st.cache_data resolve isso: na primeira chamada, a função executa normalmente e armazena o resultado em cache. Nas chamadas seguintes, o Streamlit verifica se os argumentos mudaram — se não mudaram, retorna o resultado cacheado instantaneamente.

🗄️
cache_data vs cache_resource

Use @st.cache_data para dados serializáveis (DataFrames, listas, dicts). Use @st.cache_resource para recursos não-serializáveis (conexões de banco de dados, modelos de ML carregados em memória).

Limpeza de dados — por que importa no dashboard

1

Identificar NaN

A Daniela não tem idade e o Eduardo não tem salário cadastrado. Valores ausentes causam erros nos cálculos de métricas e nos gráficos.

2

Preencher valores ausentes

Usamos a média para a idade (valores contínuos) e a mediana para salário (mais robusta a outliers). Outras estratégias: preencher com 0, com o valor da linha anterior, ou remover a linha.

3

Feature Engineering

Criamos novas colunas úteis para análise: salario_anual, ano_contratacao e categoria_salario. Essas transformações são feitas uma vez aqui, e ficam disponíveis em todo o app.

A Porque o Python exige isso para usar pandas
B Para evitar recarregar os dados a cada interação do usuário
C Porque funções rodam mais rápido que código no corpo do script
D Para poder usar o decorator @st.cache_resource

Sidebar e Filtros Interativos

A sidebar é o painel lateral que aparece à esquerda do app. É o lugar ideal para colocar controles e filtros, mantendo o espaço principal para os dados e visualizações.

PYTHON — app.py 
st.sidebar.header("🔎 Filtros")

# Widget 1: Seleção múltipla de cidades
cidades = st.sidebar.multiselect(
    "Selecione a cidade",
    options=df["cidade"].unique(),     # todas as opções disponíveis
    default=df["cidade"].unique()      # pré-selecionado: todas
)

# Widget 2: Slider de faixa de valores
faixa_salario = st.sidebar.slider(
    "Faixa salarial",
    float(df["salario"].min()),       # valor mínimo
    float(df["salario"].max()),       # valor máximo
    (float(df["salario"].min()), float(df["salario"].max()))  # valor inicial (tuple = range)
)

# ── APLICAR OS FILTROS ────────────────────────────
df_filtrado = df[
    (df["cidade"].isin(cidades)) &
    (df["salario"] >= faixa_salario[0]) &
    (df["salario"] <= faixa_salario[1])
]

🔑
O padrão central do Streamlit

Todo widget retorna um valor que você armazena em uma variável. Você usa essa variável para filtrar, calcular ou exibir dados. Quando o usuário muda o widget, a variável muda, o script re-executa, e o dashboard atualiza. É simples assim.

Prefixo st.sidebar.

Qualquer elemento do Streamlit pode ser colocado na sidebar adicionando st.sidebar. antes. Funciona com textos, widgets, gráficos — qualquer coisa. Sem o prefixo, o elemento aparece na área principal.

PYTHON — COMPARAÇÃO 
# Na área principal
opcao = st.selectbox("Cidade", ["SP", "RJ"])

# Na sidebar
opcao = st.sidebar.selectbox("Cidade", ["SP", "RJ"])

Colunas e KPIs com st.metric()

Dashboards profissionais sempre começam com os números mais importantes em destaque — os KPIs (Key Performance Indicators). O Streamlit tem o st.metric() e o sistema de colunas para criar esse layout de forma limpa.

PYTHON — app.py 
# Criar 3 colunas de largura igual
col1, col2, col3 = st.columns(3)

# Colocar um metric em cada coluna
col1.metric("💰 Salário Médio", f"R$ {df_filtrado['salario'].mean():.2f}")
col2.metric("👥 Total Funcionários", df_filtrado.shape[0])
col3.metric("📈 Salário Máximo", f"R$ {df_filtrado['salario'].max():.2f}")

Sistema de Colunas

st.columns() divide o espaço horizontal disponível em colunas. Você pode passar um número inteiro para colunas iguais, ou uma lista de pesos para colunas proporcionais:

PYTHON — EXEMPLOS DE COLUNAS 
# 3 colunas iguais
c1, c2, c3 = st.columns(3)

# 2 colunas: esquerda 2x maior que direita
c_principal, c_lateral = st.columns([2, 1])

# 4 colunas com proporções customizadas
a, b, c, d = st.columns([3, 1, 1, 2])

# Usar uma coluna como contexto
with c1:
    st.write("Conteúdo da coluna 1")
    st.bar_chart(dados)

O poder do st.metric()

O st.metric() exibe um valor com indicador de variação (delta), muito útil para mostrar crescimento ou queda em relação a um período anterior:

PYTHON — metric com delta 
st.metric(
    label="Receita Mensal",
    value="R$ 42.500",
    delta="+ R$ 3.200",        # aparece em verde com seta ↑
    delta_color="normal"    # "normal", "inverse", "off"
)
🎯
Dica de design

Repare que os KPIs do nosso app usam df_filtrado, não df original. Isso significa que os números se atualizam automaticamente quando o usuário muda os filtros na sidebar. Esse é o padrão reativo fundamental do Streamlit.

Exibindo Tabelas de Dados

O Streamlit oferece três formas de exibir dados tabulares, cada uma com seu propósito:

PYTHON — app.py 
st.subheader("📋 Dados")

# dataframe() — interativo (recomendado)
st.dataframe(
    df_filtrado,
    use_container_width=True   # ocupa toda a largura disponível
)

Comparativo: as 3 formas de exibir tabelas

Função Interativo? Editável? Quando usar
st.dataframe() ✅ Sim ❌ Não Exploração de dados — sorting, scroll, search
st.table() ❌ Não ❌ Não Tabelas estáticas pequenas, relatórios
st.data_editor() ✅ Sim ✅ Sim Quando o usuário precisa editar dados
💡
use_container_width=True

Esse parâmetro faz o componente se expandir para ocupar toda a largura do container onde está. É uma boa prática para dashboards no layout "wide", especialmente quando combinado com colunas.

Visualizações e Gráficos

Streamlit tem gráficos nativos rápidos e também suporte completo às bibliotecas mais poderosas do ecossistema Python.

PYTHON — app.py 
st.subheader("📊 Análises")

col1, col2 = st.columns(2)

# Gráfico 1: Média salarial por cidade
media_cidade = df_filtrado.groupby("cidade")["salario"].mean()
col1.bar_chart(media_cidade)

# Gráfico 2: Distribuição por categoria
categoria = df_filtrado["categoria_salario"].value_counts()
col2.bar_chart(categoria)

Gráficos nativos do Streamlit

Os gráficos nativos aceitam diretamente um DataFrame ou Series pandas, o que os torna rápidos de usar:

FunçãoTipo de gráfico
st.bar_chart()Barras verticais
st.line_chart()Linhas (séries temporais)
st.area_chart()Área preenchida
st.scatter_chart()Dispersão (x, y)
st.map()Mapa com lat/lon

Quando usar Plotly, Altair ou Matplotlib?

Os gráficos nativos são convenientes mas limitados em customização. Para dashboards mais elaborados, use bibliotecas externas:

PYTHON — Plotly Express 
import plotly.express as px

fig = px.bar(
    df_filtrado,
    x="cidade",
    y="salario",
    color="categoria_salario",
    title="Salário por Cidade e Categoria",
    barmode="group"
)

st.plotly_chart(fig, use_container_width=True)
🏆
Recomendação para dashboards em produção

Use plotly.express como biblioteca principal. Os gráficos são interativos (zoom, hover, download), bonitos por padrão e se integram perfeitamente com Streamlit via st.plotly_chart(). Os gráficos nativos do Streamlit são ótimos para protótipos rápidos.

A Nenhuma, os dois são equivalentes
B bar_chart é mais rápido de renderizar no browser

Tabela Dinâmica (Pivot Table)

Uma tabela dinâmica (pivot table) permite cruzar duas dimensões e agregar uma métrica. É o coração de qualquer análise dimensional.

PYTHON — app.py 
pivot = pd.pivot_table(
    df_filtrado,
    values="salario",           # ← métrica a calcular
    index="cidade",             # ← linhas
    columns="categoria_salario", # ← colunas
    aggfunc="mean"             # ← função: "mean", "sum", "count", "max"
)

st.dataframe(pivot)

O resultado é um DataFrame onde as linhas são as cidades, as colunas são as categorias salariais (Alto, Médio, Baixo) e os valores são a média de salário de cada combinação.

📌
Pivot Table = Cruzamento de dados

Pergunta que a pivot table responde: "Qual é o salário médio por cidade E por categoria?". Sem a pivot, você teria que fazer dois groupby separados. Com ela, você vê tudo de uma vez.

Download de Dados e Upload de Arquivos

Um dashboard útil permite ao usuário exportar os dados filtrados e também trazer seus próprios dados para análise. Vamos ver como implementar os dois.

Download — exportar CSV filtrado

PYTHON — app.py 
# 1. Converter o DataFrame em bytes CSV
csv = df_filtrado.to_csv(index=False).encode("utf-8")

# 2. Exibir o botão de download
st.download_button(
    label="Baixar CSV",
    data=csv,
    file_name="dados_filtrados.csv",
    mime="text/csv"
)

O st.download_button() gera um link de download no browser. O dado precisa estar em bytes — por isso o .encode("utf-8") no final. O parâmetro mime informa ao browser o tipo de arquivo.

Upload — carregar CSV do usuário

PYTHON — app.py 
st.sidebar.subheader("📂 Upload de CSV")

uploaded_file = st.sidebar.file_uploader(
    "Envie um CSV",
    type=["csv"]    # lista de extensões permitidas
)

# O arquivo só existe se o usuário fez o upload
if uploaded_file:
    df_upload = pd.read_csv(uploaded_file)
    st.write("📄 Dados carregados:")
    st.dataframe(df_upload)
⚠️
Padrão if/else para upload

Sempre use if uploaded_file: antes de processar o arquivo. Quando nenhum arquivo foi enviado, uploaded_file é None, e chamar pd.read_csv(None) causaria um erro.

Outros formatos suportados

PYTHON — Formatos comuns 
# CSV
df = pd.read_csv(uploaded_file)

# Excel
df = pd.read_excel(uploaded_file)

# JSON
df = pd.read_json(uploaded_file)

# Imagem (para ML)
from PIL import Image
img = Image.open(uploaded_file)
st.image(img)

Consolidando o Conhecimento

Arquitetura completa do app

Vamos revisitar o app como um todo e entender o fluxo de dados:

1

set_page_config + title

Configuração inicial. set_page_config deve ser sempre a primeira chamada st. do script.

2

@st.cache_data → carregar_dados()

Dados são carregados, limpos e enriquecidos uma única vez. O cache garante que isso não se repita a cada interação.

3

st.sidebar — Widgets de filtro

O usuário interage com multiselect e slider. Os valores retornados são armazenados em cidades e faixa_salario.

4

df_filtrado = df[condições]

O DataFrame original é filtrado com base nos valores dos widgets. Todo o restante do app usa df_filtrado.

5

KPIs → Tabela → Gráficos → Pivot → Download

Componentes visuais exibem os dados filtrados. Cada um reage automaticamente quando df_filtrado muda.
